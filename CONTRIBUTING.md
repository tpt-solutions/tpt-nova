# Contributing to TPT Nova

Thanks for your interest in TPT Nova! This document covers the local gates that
CI enforces so your PR is green on the first try.

## Repository layout

TPT Nova is a Cargo workspace. Each capability lives in its own crate under
`crates/` (see the table in `README.md`). Cross-crate APIs are kept explicit via
workspace path dependencies — there are no hidden global singletons.

## Before you push

Run these locally; CI runs the same commands (and fails the build on any
warning, because clippy is invoked with `-D warnings`):

```sh
cargo fmt --all -- --check      # formatting (run `cargo fmt --all` to fix)
cargo clippy --workspace --all-targets -- -D warnings
cargo build --workspace --all-targets
cargo test --workspace
```

`cargo clippy --workspace --all-targets -- -D warnings` turns every lint into a
hard error, so unused imports, dead code, and clippy suggestions will block a
merge. Treat warnings as errors locally.

## Commit & PR conventions

- Keep commits focused; the repo history uses short imperative subjects
  (e.g. `Add editor viewport gizmo wiring`) and explains *why*, not just *what*.
- Do not commit secrets, downloaded model weights, or large binaries. The only
  committed binary assets are the tiny samples under `assets/` (generated by
  `cargo run -p nova-ingest --example gen_sample_assets`).
- Update `todo.md` checkboxes when you close a roadmap item; keep the audit
  notes honest.

## Testing expectations

Every crate ships `#[test]` coverage (see the "Testing Coverage" section of
`todo.md`). When you add a feature, add a unit or integration test for it:

- Pure logic (ECS math, editor panels, telemetry, RAG, overlay) is fully
  unit-testable without a window or GPU.
- GPU-only paths (the `wgpu` render loop) are covered by `build_ui_vertices` /
  camera-math unit tests; do not require a display in CI.
- Deterministic behavior (fixed-timestep, seeded RNG, physics) should assert
  exact, reproducible output.

## Untrusted-input surfaces

TPT Nova is AI-native, so several inputs come from outside the engine. When you
touch one of these, be especially careful:

- **`nova-agent-api`** — parses a control file written by an external agent.
  File size is capped and the protocol version is checked.
- **`nova-scripting-embedded`** — runs Rhai scripts with an explicit capability
  boundary; `eval`/`import` are disabled and operation/memory limits are set.
- **`nova-export`** — unpacks `.novapack` archives; length prefixes are
  bounds-checked against remaining bytes before any allocation, and entry names
  are rejected if they try path traversal.

If you add a new untrusted-input path, add a regression test that feeds it a
hostile input (oversized length, traversal name, bad magic, etc.).
